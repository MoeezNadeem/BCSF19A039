Git Teaching website

  1. Git init

    2. Command : "git init"

    Explaination : Git init commmand is used to initialize an empty git repository. This command creates a hidden ".git" folder in our current working directory. Most other Git commands are not available outside of an initialized repository, so this is usually the first command you'll run in a new project.

  2. Git Status

    3. Command : "git status"

    Explaination : The git status command displays the state of the working directory and the staging area. It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git. Status output does not show you any information regarding the committed project history.

  3. Git Add

    4. Command : "git add"

    Explaination : Git add commmand is used to add untracked files to the staging index.The git add command adds a change in the working directory to the staging area. It tells Git that you want to include updates to a particular file in the next commit. However, git add doesn't really affect the repository in any significant way changes are not actually recorded until you run git commit. "git add . " command is used to add all the files in current working directory to the staging index.

  4. Git Commit

    5. Command : "git commit -m “[ Type in the commit message]” "

    Explaination : This command records or snapshots the file permanently in the version history.

  5. Git branch

    6. Command : "git branch"

    Explaination : This command lists all the local branches in the current repository.

  6. Git Push

    7. Command : "git push"

    Explaination : This command sends the committed changes of master branch to your remote repository.

  7. Git Pull

    8. Command : "git pull [repository link]"

    Explaination : This command fetches and merges changes on the remote server to your working directory.

  8. Git checkout

    9. Command : "git checkout [branchname]"

    Explaination : This command is used to switch from one branch to another.

  9. Git log

    10. Command : "git log"

    Explaination : This command is used to list the version history for the current branch.

  10. Git config

    11. Command : git config user.name “[name]”
    git config global user.email “[email address]”

    Explaination : This command sets the author name and email address respectively to be used with your commits.

  11. Git clone

    12. Command : git clone [url]

    Explaination : This command is used to obtain a repository from an existing URL.
